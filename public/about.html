Mismatched CIs dashboard provides users the ability to identify and analyze mismatched server CI owner and billing data, in an effort to eliminate misaligned billing charges.

Development and Deployment

Application Information



Story Board

Need to be able to log into app and manage CI data based on user credentials (email & password) • Should be able to upload a CSV file containing CI, CI Owner, Owning and Billing Department data • Upon data import, filter out CI data that does not have a match between CI department owner and department billed by, querying DB by user and department ID • Display only missmatched CI data as a list on HTML dashboard • Display a graph of missmatched CI data by Department • Add funtionality to notify / email the owner of mis-alinged CI data to correct in the CMDB

Technology

Mismatched CI Dashboard is a web application using a Node and Express Web Server, backed by a MySQL Database. Sequelize is being used as the interface between the two system (ORM). The application is deployed on a public cloud platform, Heroku, for the Production environment. User authorization and data access is achieved using JWT. The middleware, HelmetJS, has been implemented to add application security by setting secure HTTP headers and secure content policies, amongst others. To meet new technology requirements, D3 (JavaScript library) is being used to produce dynamic, interactive data visualizations. Custom NPM "scripts" can be found in Package.js, and API Endpoint Testing is performed using Chai for Test Driven Development (from the terminal -> npm run windowstest)

Functionality (MVP1)

User restricted access / login required • Import CI data from CSV file containing the "department" and "owned_by_department" fields used to filter on required data. The data model can be seen in the Development & Deployment section above • View CI records where there is a mismatch between owning department and billing department data • View Bar and Pie chart graphed CI data by "User", and by "Department" • Launch Service Now from the Dashboard to make CMBD updates, gather new CSV reports, etc • Add new users to tha application via the User Profile page • NPM scripts available for package management and cleanup
